# 🧪 Tree Popup 功能测试指南

## ✅ 功能已实现

在"分解结构"树中点击节点时，会弹出一个包含以下选项的窗口：
1. **继续拆解** - 和圆形节点的继续拆解逻辑完全一致
2. **生产规划** - 保留原有的跳转/弹窗逻辑
3. **关闭按钮** - 右上角的 × 可以关闭弹窗

## 🎮 手动测试步骤

### 方法 1: 完整流程测试

1. **访问应用**
   ```
   http://localhost:3000
   ```

2. **注册/登录**
   - 如果没有账号，先注册
   - 如果有账号，直接登录

3. **识别物品**
   - 在 setup 页面选择"文字"输入
   - 输入物品名称（例如："智能手机"、"笔记本电脑"、"四足机器人"）
   - 点击"开始识别"
   - 等待识别完成（约5-8秒）

4. **选择生产模式**（可选，但推荐）
   - 点击"生产模式"按钮
   - 这样才会有"生产规划"选项

5. **开始拆解**
   - 点击"开始拆解"按钮
   - 等待跳转到 canvas 页面
   - 再次点击"开始拆解"按钮
   - 等待拆解完成（约10-15秒）

6. **测试树节点弹窗**
   - 在左侧的"分解结构"树中，点击任意节点
   - 应该会弹出一个窗口，包含：
     - ✅ 节点名称和描述
     - ✅ "继续拆解"按钮
     - ✅ "生产规划"按钮（如果选择了生产模式）
     - ✅ 关闭按钮（右上角 ×）

7. **测试按钮功能**
   - 点击"继续拆解"：应该开始拆解该节点，并在树中添加子节点
   - 点击"生产规划"：应该跳转到生产分析页面或显示生产规划弹窗
   - 点击关闭按钮：弹窗应该关闭

## 🐛 已知问题和解决方案

### 问题 1: 弹窗没有出现

**可能原因：**
- 点击了折叠/展开按钮而不是节点本身
- 节点是原材料（原材料节点不显示弹窗）

**解决方案：**
- 确保点击节点的主体部分，而不是左侧的折叠/展开按钮
- 选择非原材料节点进行测试

### 问题 2: 识别失败

**可能原因：**
- 没有配置 API 密钥
- API 服务不可用

**解决方案：**
- 检查 `.env.local` 文件中的 API 密钥配置
- 确保 API 服务正常运行

### 问题 3: 拆解失败

**可能原因：**
- API 调用失败
- 网络问题

**解决方案：**
- 检查浏览器控制台的错误信息
- 刷新页面重试

## 🔧 调试技巧

1. **打开浏览器开发者工具**
   - 按 F12 或右键 -> 检查
   - 查看 Console 标签页的错误信息

2. **检查元素**
   - 使用元素选择器（Ctrl+Shift+C）
   - 检查弹窗的 DOM 结构
   - 确认弹窗是否被正确渲染

3. **查看网络请求**
   - 切换到 Network 标签页
   - 检查 API 请求是否成功
   - 查看请求和响应的内容
